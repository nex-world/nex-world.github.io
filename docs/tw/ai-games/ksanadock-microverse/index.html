AI 遊戲推薦KsanaDock · Microverse In Box

import { Callout } from ‘nextra/components’

在自己的電腦裡養一個AI社會——《時空碼頭 · 盒中小世界》

8個AI角色在你的電腦裡談戀愛、搞事業、建立完整的社會關係——而你，是俯瞰這個世界的上帝。

👉 Steam 頁面（盒中小世界） 
👉 官網（時空碼頭） 
👉 GitHub 開源倉庫 

這到底是什麼？

2023 年，史丹佛大學發表了一篇著名的論文——Generative Agents: Interactive Simulacra of Human Behavior。研究者讓 25 個 AI 角色生活在一個小鎮裡，發現他們會自主社交、組織活動、甚至策劃一場情人節派對。這就是後來廣為人知的 「史丹佛 AI 小鎮」。

但那個實驗需要一個複雜的 Python 後端，一般人很難在自己的電腦上跑起來。

《盒中小世界》（英文名：Microverse In Box）的出現，改變了這一點。它把整個 AI 社會放進了一個 Godot 4 遊戲引擎 構建的桌面應用裡——不需要額外部署 Python 服務，不需要 LangChain，下載打開、填上 API Key，你的 AI 小鎮就開始運轉了。

**遊戲定位**：上帝模擬沙盒（God-simulation Sandbox）。你並非作為角色參與其中，而是以俯視視角觀察和干預這個 AI 社會的演化。

時空碼頭是誰？

時空碼頭（KsanaDock） 是一個由 個人獨立開發者 發起的 AI 原生遊戲專案。

「Ksana」源自佛教術語「剎那」（क्ण, kṣaṇa），意為極短的時間單位——取名「時空碼頭」，寓意每一個剎那都在發生故事，而這裡是故事的碼頭。

開發者（也被社群稱為「咪噢喵」或「顧伊凡 YGY」）在 2025 年 6 月將初版 Demo 開源到 GitHub 後，短短 3 天內就收穫了 700+ Stars，隨後迅速攀升至 2300+ Stars。在知乎上，他寫下《一個人能做出什麼開源專案？》的回答，講述了一個獨立開發者如何用 Godot 4 和 LLM API，從零搭建一套多智慧體 AI 社交模擬系統。

這是一個關於「一個人造了一個世界」的故事。

遊戲核心機制

🎭 8 個 AI 角色，各懷心事

遊戲預置了 8 個 AI 角色，每個都有獨立的性格、背景故事和行為模式：

角色定位性格特徵
Alice友善的專案經理善於溝通，協調團隊
Grace專業的資料分析師理性縝密，注重細節
Jack創意設計師天馬行空，不拘一格
Joe技術專家實幹派，解決問題導向
Lea市場行銷專家外向健談，洞悉人心
MonicaHR 專員細緻周到，關心他人
Stephen財務分析師謹慎保守，精打細算
Tom軟體開發工程師專注技術，偶爾社交恐懼

這些角色並非靜止的人設面板——他們會工作、交流、爭吵、合作，並在長期相處中演化出真實的社會關係。

🧠 AI 驅動的四大系統

1. 獨立思維與對話

每個角色的每一句話、每一個行動，都由大語言模型即時生成。沒有固定臺詞、沒有腳本分支——你的 API Key 叩開哪個模型（GPT-4、Claude、DeepSeek……），他們就擁有哪個模型的「智商」和「性格」。

2. 長期記憶系統

角色們會記住發生過的事。昨天 Tom 在會議上反駁了 Grace 的方案，今天 Grace 見到 Tom 時態度會微妙地冷淡；上週 Alice 幫 Jack 解了圍，Jack 會一直記得這份人情。

記憶會被持續寫入 JSON 檔案儲存在本地，即使關掉遊戲再打開，這個社會也不會「失憶」。

3. 自主任務管理

AI 角色會自主安排自己的日程：早上起床 → 吃早飯 → 去工位工作 → 午休聊天 → 下午開會 → 下班社交……他們不是被動等待玩家指令的木偶，而是有自己生活節奏的「數位生命」。

4. 環境感知與反應

角色能感知周圍發生了什麼。如果兩個人在旁邊大聲爭吵，附近角色會聽到並做出反應——可能會圍觀、勸架，或者默默走開。

🃏 卡牌系統（Steam 完整版）

即將上線的 Steam 完整版將引入 卡牌系統，讓你可以像上帝一樣直接干預這個世界：

  • 修改角色的 人設和性格
  • 調整 財富和資源分配
  • 改變 角色間的關係狀態
  • 插入 事件和劇情轉折
  • 自訂 世界運行的底層規則

想看社交恐懼的 Tom 突然變成社交達人？想看看如果 Alice 中樂透會發生什麼？一張卡牌的事。

技術特色：為什麼它值得關注？

一個包搞定一切

與史丹佛 AI 小鎮的原始實現不同，《盒中小世界》將全部邏輯放在 Godot 4 引擎內部實現：

  • ✅ 不依賴額外的 Python 服務
  • ✅ 不需要 LangChain 或類似框架
  • ✅ 下載即用，開箱即體驗
  • ✅ 支援 Windows / macOS / Linux / Android 多平臺

支援幾乎所有主流 AI 模型

玩家可以自由選擇後端模型，甚至隨時切換：

類別支援的平臺
雲端 APIOpenAI（GPT-4/3.5）、Anthropic Claude、Google Gemini、DeepSeek、字節豆包、月之暗面 Kimi
本地部署Ollama（執行 Llama、Qwen 等開源模型）

這意味著：如果你注重隱私或不想花錢，可以接 Ollama 在本地跑開源模型，完全離線執行；如果你追求角色的「智商」和對話品質，可以用 GPT-4 或 Claude 獲得最頂級的體驗。

純開源 + MIT 授權

專案採用 MIT 開源授權，可自由商用。初版 Demo 完整開源在 GitHub 上，任何人都可以下載學習、修改、甚至基於它開發自己的 AI 遊戲。

如何體驗？

方式一：GitHub 開源版（免費）

git clone https://github.com/KsanaDock/Microverse.git

用 Godot 4.3+ 打開專案根目錄，設定 AI API Key 即可執行。適合開發者、AI 愛好者和想先嚐鮮的玩家。

方式二：Steam 完整版（即將上線）

👉 加入願望清單

  • 🗺️ 上帝視角地圖編輯器
  • 📖 互動式小說製作工具

值得買嗎？

👍 如果你有以下期待，它會是你的菜：

  • 對「AI 社會模擬」這個概念充滿好奇
  • 看過史丹佛 AI 小鎮論文，想親手玩到類似的實現
  • 喜歡觀察和引導（而不是直接參與）的上帝類遊戲

  • 對開源和獨立開發有情懷

👎 以下情況可能需要再想想：

  • 期望的是畫面精美、劇情打磨充分的 3A 級商業遊戲
  • 不想自己準備 API Key 或設定模型（遊戲本身不包含 AI 模型，需要自備）
  • 對「看 AI 自己玩」這種旁觀者玩法提不起興趣

一個觀察：為什麼它讓人上癮？

如果你看過《楚門的世界》或者《西方極樂園》，應該能理解這種體驗的獨特魅力——看著螢幕裡那些像素小人忙忙碌碌，你會不自覺地開始腦補他們的故事。

「Tom 今天怎麼一個人坐在角落裡？是不是昨天跟 Joe 吵架了？」
「Alice 和 Jack 最近互動變多了啊……是不是有什麼進展？」

這種 投射心理 是人類與生俱來的本能。當你知道螢幕裡的每一個角色都有獨立的 AI 大腦、真實的記憶、自主的行動邏輯時，那種「他們在真實地活著」的沉浸感，遠超傳統的腳本式 NPC。

結語

如果說《歷史模擬器：崇禎》讓你看到了 AI 如何驅動一個宏大敘事的王朝興衰，那麼《盒中小世界》則是另一個方向的極致——它讓你在微觀尺度上，親眼見證一個 AI 社會的日常流動。

一個關乎國運，一個關乎人心。兩者放在一起，恰好構成了 AI 原生遊戲在「宏大」與「日常」兩個維度上的完整拼圖。
