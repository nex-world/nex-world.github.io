AIゲームKsanaDock · Microverse In Box

自分のパソコンでAI社会を育てる——KsanaDock：箱の中の小世界（Microverse In Box）

8体のAIキャラクターがあなたのパソコンの中で恋をし、仕事をし、完全な社会関係を築いていく——そしてあなたは、この世界を見下ろす神だ。

👉 Steamページ（箱の中の小世界） 
👉 公式サイト（時空埠頭） 
👉 GitHubオープンソースリポジトリ 

これは何か？

2023年、スタンフォード大学は画期的な論文を発表しました——Generative Agents: Interactive Simulacra of Human Behavior。研究者たちは25体のAIキャラクターを小さな町に住まわせ、彼らが自発的に交流し、イベントを企画し、さらにはバレンタインデーのパーティーを計画することを発見しました。これが後に広く知られるようになった**「スタンフォード AI タウン」**です。

しかし、その実験には複雑なPythonバックエンドが必要で、一般の人が自分のパソコンで動かすのは困難でした。

「箱の中の小世界」（英名：Microverse In Box）の登場は、それを変えました。AI社会全体をGodot 4ゲームエンジンで構築されたデスクトップアプリケーションに詰め込みました——追加のPythonサービスのデプロイは不要、LangChainも不要。ダウンロードして開き、APIキーを入力するだけで、あなたのAIタウンが動き始めます。

🏗️

ゲームの位置づけ：神視点シミュレーションサンドボックス。プレイヤーはキャラクターとして参加するのではなく、見下ろし視点でこのAI社会の進化を観察し、介入します。

時空埠頭（KsanaDock）とは？

時空埠頭（KsanaDock） は、個人の独立した開発者によって立ち上げられたAIネイティブゲームプロジェクトです。

「Ksana」は仏教用語の「刹那」（क्ण, kṣaṇa）に由来し、極めて短い時間単位を意味します——「時空埠頭」という名前には、一瞬一瞬に物語が生まれ、その物語が集まる埠頭（ドック）であるという意味が込められています。

開発者（コミュニティでは「ミオニャン」や「グー・イーファン YGY」の名でも知られています）は2025年6月に初版デモをGitHubでオープンソース化しました。わずか3日間で700以上のスターを獲得し、その後2300以上のスターにまで急成長しました。知乎（Zhihu）では、「一人でどんなオープンソースプロジェクトが作れるのか？」という質問に答え、一人の開発者がGodot 4とLLM APIを使ってゼロからマルチエージェントAIソーシャルシミュレーションシステムを構築した物語を綴っています。

これは「一人の人間が一つの世界を創った」物語です。

ゲームの中核メカニズム

🎭 8体のAIキャラクター、それぞれの事情

ゲームには8体のAIキャラクターがプリセットされており、それぞれに独自の性格、背景ストーリー、行動パターンがあります：

キャラクター役割性格特性
Alice友好的なプロジェクトマネージャーコミュニケーション上手、チームをまとめる
Graceプロフェッショナルなデータアナリスト理性的で緻密、細部にこだわる
Jackクリエイティブデザイナー自由奔放、型にはまらない
Joe技術専門家実践的、問題解決志向
Leaマーケティング専門家外向的で話し好き、人心に精通
MonicaHRスペシャリスト細やかで思いやりがあり、他人を気遣う
Stephen財務アナリスト慎重で保守的、倹約家
Tomソフトウェア開発エンジニア技術に没頭、時々社交恐怖

これらのキャラクターは静止したプロフィールパネルではありません——彼らは仕事をし、会話し、議論し、協力し、長い付き合いの中でリアルな社会関係を築いていきます。

🧠 AI駆動の4大システム

1. 独立した思考と対話

各キャラクターのすべての言葉と行動は、大規模言語モデルによってリアルタイム生成されます。固定された台詞もスクリプト分岐もありません——あなたのAPIキーが開くモデル（GPT-4、Claude、DeepSeek……）が、そのまま彼らの「知能」と「性格」になります。

2. 長期記憶システム

キャラクターたちは起きた出来事を覚えています。昨日Tomが会議でGraceの提案に反論したら、今日GraceがTomに会ったときの態度は微妙に冷たくなる。先週AliceがJackを助けたら、Jackはその恩義をずっと覚えている。

記憶は継続的にJSONファイルに書き込まれてローカルに保存されるので、ゲームを閉じて再開しても、この社会は「記憶喪失」になりません。

3. 自律的なタスク管理

AIキャラクターは自らのスケジュールを自律的に組み立てます：朝起床 → 朝食 → 職場で仕事 → 昼休みに雑談 → 午後は会議 → 退社後に交流……彼らはプレイヤーの指示を待つ操り人形ではなく、自分自身の生活リズムを持つ「デジタル生命」です。

4. 環境認識と反応

キャラクターは周囲で起きていることを感知できます。もし二人が近くで大声で言い争っていたら、近くのキャラクターはそれを聞いて反応します——野次馬になるか、仲裁に入るか、あるいは静かに立ち去るか。

🃏 カードシステム（Steam完全版）

まもなくリリースされるSteam完全版では、カードシステムが導入され、神のようにこの世界に直接介入できるようになります：

  • キャラクターの設定や性格を変更
  • 富や資源の配分を調整
  • キャラクター間の関係性を変更
  • イベントやストーリーの展開を挿入
  • 世界の根底にあるルールをカスタマイズ

社交恐怖症のTomが突然社交的になるのを見たい？Aliceが宝くじに当たったら何が起きる？たった一枚のカードで実現します。

技術的特徴：なぜ注目に値するのか

一切合切を一つのパッケージに

スタンフォードAIタウンのオリジナル実装とは異なり、「箱の中の小世界」はすべてのロジックをGodot 4エンジン内部で実装しています：

  • ✅ 追加のPythonサービス不要
  • ✅ LangChainや類似フレームワーク不要
  • ✅ ダウンロードしてすぐに体験可能
  • ✅ Windows / macOS / Linux / Androidのマルチプラットフォーム対応

ほぼすべての主要AIモデルに対応

プレイヤーは自由にバックエンドモデルを選択でき、いつでも切り替えられます：

カテゴリ対応プラットフォーム
クラウドAPIOpenAI（GPT-4/3.5）、Anthropic Claude、Google Gemini、DeepSeek、ByteDance Doubao、Moonshot AI Kimi
ローカルデプロイOllama（Llama、Qwenなどのオープンソースモデルを実行）

つまり：プライバシーを重視する方やお金をかけたくない方は、Ollamaを使ってローカルでオープンソースモデルを実行でき、完全オフラインで動作します。キャラクターの「知能」や対話の質を追求するなら、GPT-4やClaudeで最高の体験が得られます。

純粋オープンソース + MITライセンス

プロジェクトはMITオープンソースライセンスを採用しており、商用利用も自由です。初版デモはGitHubで完全にオープンソース化されており、誰でもダウンロードして学習、修正、さらにはそれを基に独自のAIゲームを開発することができます。

方法1：GitHubオープンソース版（無料）

git clone https://github.com/KsanaDock/Microverse.git

Godot 4.3+でプロジェクトルートを開き、AI APIキーを設定すれば実行できます。開発者、AI愛好家、いち早く試してみたいプレイヤーに最適です。

買う価値はあるか？

👍 こんな方におすすめ：

  • 「AI社会シミュレーション」というコンセプトに好奇心がある
  • スタンフォードAIタウンの論文を読んで、同じようなものを実際にプレイしてみたい
  • 観察と誘導（直接参加ではない）を楽しむ神視点ゲームが好き
  • 異なるAIモデルで比較実験をしたい——同じシナリオでGPT-4とDeepSeekがどう違う社会進化を見せるか？
  • オープンソースとインディー開発に愛着がある

👎 こんな方は再考の余地あり：

  • グラフィックが美しく、ストーリーが練り込まれた3A級の商業ゲームを期待している
  • 自分でAPIキーを用意したり、モデルを設定したりしたくない（ゲーム自体にAIモデルは含まれておらず、自分で用意する必要があります）
  • 「AIが勝手に遊ぶのを見る」という傍観者的な玩法に興味が持てない

ひとつの観察：なぜハマるのか？

もしあなたが『トゥルーマン・ショー』や『ウエストワールド』を見たことがあるなら、この体験のユニークな魅力が理解できるはずです——画面の中のドット絵の小人たちが忙しそうに動き回るのを見ていると、自然と彼らの物語を想像してしまうのです。

「どうしてTomが今日は一人で隅っこに座ってるんだろう？昨日Joeと喧嘩でもしたのかな？」 「AliceとJack、最近やけに交流が多いね……何か進展でもあったのかな？」

こうした投影心理は、人間に生まれつき備わった本能です。画面の中のキャラクター一人ひとりに独立したAIの頭脳、本物の記憶、自律的な行動ロジックがあると知ったときの、「彼らは本当に生きている」という没入感は、従来のスクリプト型NPCをはるかに超えます。
